需求阶段

从「我觉得」到「被验证的假设」

AI 作为调研与推理引擎，PM 退为问题定义者。一条可落地的七步链路：

  • 问题对齐：业务目标 → 可度量指标，识别信息缺口
  • 信息收集：聚类埋点 / 工单 / 竞品，输出痛点证据
  • 方案共创：每个机会出 MVP / 体验 / AI 增强三版
  • 最小验证：AI 模拟用户访谈、Prompt 级 MVP
需求一次成链路返工率 ↓
架构阶段

AI 辅助识别关键决策点

输入需求与现有技术栈，AI 拆解功能 / 非功能要求，标出需架构师拍板的关键决策并给出选项分析。

  • 租户隔离：共享库+tenant_id 还是独立库？
  • LLM 集成：绑定单一 还是 可插拔？
  • 检索策略：纯向量 还是 混合召回？
  • 成本与时效：热冷分离 / 异步批处理
决策点前置选型可追溯

QPS 282 → 286（稳定） /v4/embeddings QPS 103 → 112（后半段起量）

需求阶段 · 七步落地法

每步都有「PM 做什么 / AI 做什么 / 产出物」

  • ① 问题对齐：目标拆指标、识别信息缺口 → 《问题定义 One-pager》
  • ② 信息收集：聚类痛点、提炼关键路径 → 《痛点证据表》
  • ③ 机会空间：打分排序给优先级 → 《机会评估矩阵》
  • ④ 方案共创：多版本方案草图 → 《方案对比表》
  • ⑤ 最小验证：AI 模拟访谈 / 日志回放 → 《验证结论摘要》
  • ⑥ PRD 自动化：拼装初稿 + 评审清单 → 《PRD v1.0》
  • ⑦ 设计稿联动：生成 IA + Wireframe Brief → 可导入 Figma
最终交付 PRD v1.0 + Wireframe，同源生成、信息一致、可追溯。
架构阶段 · 关键决策识别

AI 把模糊需求变成「待拍板清单」

  • 需求拆解：自动分出功能性 / 非功能性要求
  • 租户隔离：共享库+tenant_id vs 独立库（含升级预留）
  • LLM 集成：建议可插拔结构，作为核心竞争力
  • 检索架构：异步批量向量化 + 混合召回 + Re-rank
  • 性能 / 成本：p95 延迟目标、热冷分离、按需扩容
  • 评估体系：标准指标（RAGAS）+ 业务自定义，定期自动化
基于现有技术栈选型，优先复用已有组件，必要时才引入新中间件。

如何把这些能力用起来

GLM 模型提供三种接入方式，按场景与数据安全要求灵活组合；能力可通过工具与 MCP 扩展，并可沉淀企业专属知识。

方式一 · API 直连
最快验证

OpenAI 兼容接口，几行代码调用 GLM-4.6 / 4.7 / 5.2，适合自研应用、数据代码生成、虚拟用户后端等场景。

方式二 · 编码工具集成
嵌入研发流

在 Claude Code、Cursor 等主流编码工具中配置 GLM，直接用于研发全链条的 AI Coding。

方式三 · 私有化部署
数据不出域

政务、金融、能源等敏感行业，模型私有化部署 + 企业知识库接入，深度适配业务规则，越用越懂业务。
