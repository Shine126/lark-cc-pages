五个问题的回答

> 说明：这五题按 API/企业客户 口径作答（安全问卷的通常语境）。Z.ai 对「API 服务」和「消费者 Chat 产品」是两套不同数据政策，下面会标出来。

1️⃣ 数据加密方式

结论：传输加密有据可查；存储加密政策级承诺，但公开文档未点名具体算法（AES-256/TLS 版本号需正式 DPA/安全白皮书确认）。

- 传输层：API 端点全 HTTPS（https://api.z.ai/api/paas/v4/...），即 TLS 加密通道。企业版 Coding Plan 承诺「更高标准的传输与存储加密」。

- 存储层：国内 BigModel 隐私政策明确「敏感信息加密存储」「数据分类分级保护」「身份鉴别 + 访问控制」；海外 Z.ai 称采用「commercially reasonable technical, administrative, and physical security measures」。

- ⚠️ 缺口：公开政策未写明具体加密算法（AES-256）、密钥管理（KMS/CMEK）、TLS 版本。客户若要填安全问卷，这块要找商务/安全团队拿正式安全白皮书或 DPA，不能凭公开文档臆测。

📄 来源：[BigModel 隐私政策 §四](https://docs.bigmodel.cn/cn/terms/privacy-policy)（2025-05-20 生效）｜[Z.ai Privacy Policy §6](https://docs.z.ai/legal-agreement/privacy-policy)（2025-09-29 更新）

2️⃣ 数据保留周期与日志策略

结论：API 服务的输入/输出内容默认零保留（实时处理即删）；元数据/日志按服务必需 + 法律义务保留。

- API 内容数据：Z.ai DPA §4(b) 原文——「The Company do not store any of the content the Customer or its End Users provide or generate... This information is processed in real-time... and is not saved on our servers.」（输入输出不落盘，实时处理后即弃）。

- 其他客户数据：DPA §4(c) 临时存储以提供服务或满足法律要求，服务终止后删除。

- 日志：Z.ai 收集 Log Information（日志文件、错误堆栈、调用性能），用于运维/安全/反欺诈；保留期随数据量/类型/敏感度及法律义务而定（消费者侧：账号存续期内保留）。

- 数据驻留：国内 BigModel 数据存中国境内（公有云）；海外 Z.ai 主要在新加坡处理。

📄 来源：[Z.ai DPA §4](https://docs.z.ai/legal-agreement/privacy-policy)（API 数据处理附录）｜[BigModel 隐私政策 §五](https://docs.bigmodel.cn/cn/terms/privacy-policy)

3️⃣ IAM / RBAC 权限管理

结论：内部员工 RBAC + 最小授权；客户侧 API Key 鉴权 + 团队/企业版的成员与组织管理。

- 内部访问控制：BigModel「部署访问控制机制，确保只有授权人员才可访问客户数据」+ 数据分类分级 + 身份鉴别。

- 产品级 RBAC：GLM-5-Turbo 官方技术页明确「基于角色的权限管理（RBAC），确保每个 Agent 只能在授权范围内访问数据/调用工具」+ 完整审计日志。

- 客户侧：API 通过 Bearer API Key 鉴权；GLM Coding Plan 团队版/企业版提供成员管理、用量分析、组织管理控制（可按席位分配 Key、查看用量）。

📄 来源：[BigModel 隐私政策 §四(一)](https://docs.bigmodel.cn/cn/terms/privacy-policy)｜[GLM-5-Turbo 技术页](https://www.zhipuai.cn/zh/research/155)｜[Coding Plan 团队版权益](https://docs.bigmodel.cn/cn/coding-plan/team)

4️⃣ Tenant Isolation / 数据隔离

结论：三层隔离——逻辑隔离（账号/API Key 边界）+ 企业版严格隔离 + 私有部署物理隔离。

- 逻辑隔离：Z.ai API DPA 下智谱作为 Data Processor，仅按客户指令处理 Customer Data，账号间天然隔离。

- 企业版强隔离：GLM Coding Plan 企业版承诺「企业数据严格隔离」+ 传输存储加密。

- 物理隔离（最高档）：支持 Private Instance Deployment（私有实例部署）——模型部署在客户专属计算资源上，实现「数据安全、性能隔离、资源可控」。这是对隔离要求最高客户的终极方案。

- 地域隔离：国内数据不出境（存中国境内）；海外数据走新加坡，跨境走合规传输机制。

📄 来源：[Z.ai DPA](https://docs.z.ai/legal-agreement/privacy-policy)｜[私有实例部署文档](https://docs.bigmodel.cn/cn/guide/tools/model-deploy)｜[BigModel 隐私政策 §五](https://docs.bigmodel.cn/cn/terms/privacy-policy)

5️⃣ 是否使用客户数据训练（禁止=10）

结论：✅ API/企业客户数据不存储、不用于训练 —— 符合「禁止」标准，该项可得 10 分。

这是最关键的一题，有官方 DPA 白纸黑字背书：

- Z.ai API DPA §4(b) 原文：「The Company do not store any of the content the Customer or its End Users provide or generate... This information is processed in real-time to provide the API Service and is not saved on our servers.」

→ 输入输出连落盘都不落盘，自然不可能用于训练。

- 模型训练数据来源：Z.ai 隐私政策明确模型训练用的是「Public Information（互联网公开数据）」+ 匿名化/聚合数据，不是客户业务数据。

- 国内 Coding Plan 企业版：公开报道（品玩/OSCHINA）确认智谱承诺「数据不用于训练」「企业数据隔离与加密」。

⚠️ 重要边界（避免被客户追问时翻车）：

- 上述「不训练」承诺仅适用于 API/企业服务。

- Z.ai / Chat 的消费者侧 Chat 产品，隐私政策允许用匿名化数据改进模型（§3「train and improve our models」）。若客户用的是消费者版而非 API，口径要区分清楚。

- 填正式安全问卷时，建议附上 DPA 原文 作为「禁止训练」的法律依据，比口头承诺更硬。
